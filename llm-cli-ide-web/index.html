名詞定義

名詞 是什麼 例子
Tool 工具 你實際操作的軟體或插件 Kiro IDE、Q CLI、Quick Suite、VS Code
LLM 大型語言模型 真正在「思考」和生成回答的 AI 模型 Claude Opus 4.5、GPT-5.2、Gemini 3 Pro

關鍵觀念

  • 工具 ≠ LLM
    Kiro IDE 是工具，Claude Opus 4.5 是 LLM，它們是不同的東西。
  • 一個工具可以連接一個或多個 LLM
    例如 VS code 允許用戶選擇來自不同公司，甚至自研的 LLM 。
  • 不同工具也能用同一個 LLM
    例如 Google Map、NotebookLM、Google Search 三個不同服務的後面，可能都是用了 Gemini 3 Pro 作為 LLM 。

常見誤解

不精準的說法 更精準的說法 為什麼
Kiro IDE 是 AI 模型 Kiro IDE 是一個使用 LLM 的軟體 Kiro IDE 是工具，不是 LLM / AI
Quick Suite 是 AI 工具 Quick Suite 是可以呼叫 LLM 的瀏覽器插件 Quick Suite 是工具，不是 LLM / AI
Q CLI 比較笨 不同工具可能使用相同的 LLM，智能不會有差距 LLM 決定智能，工具決定介面

🛠️認識 3 種工具：CLI、IDE、Web

💡 1 秒自檢 — 怎麼知道現在該用哪個工具?：
  • 你正在看網頁，想針對網頁內容提問? → 用 Web 的瀏覽器插件 (e.g. Quick Suite)
  • 你只是想快速問個問題? → 用 Web 的網頁工具 (e.g. ChatGPT, AI Studio) 或 CLI
  • 你想要進一步研究或開發專案? → 用 IDE
  • 你不知道你現在該用哪個工具? → 用 IDE

CLI

黑視窗、純鍵盤操作

適合

  • 快速問一個問題
  • 跨資料夾的檔案操作
  • 根據 Readme 安裝程式

不適合

  • 需要看完整專案結構
  • 需要來回修改多版本
  • 想用截圖作為輸入

IDE

視窗軟體、滑鼠操作

適合

  • 複雜專案開發
  • 需要來回修改多版本
  • 不確定該用什麼工具時

不適合

  • 只是快速問一個問題
  • 處理網頁上的內容

Web

網頁工具、瀏覽器插件

適合

  • 快速問答、腦力激盪
  • 翻譯或彙總網頁內容
  • 不需安裝軟體即可使用

不適合

  • 處理電腦裡的檔案
  • 需要複雜的專案開發
  • 需要大量上下文額度

Kiro IDE 跟 Kiro CLI 都有 Kiro logo，所以是同一個軟體的不同版本
