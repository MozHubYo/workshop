為什麼要學?

為什麼 AI 用起來好像沒有別人說的那麼好用?

本地端的 AI 跟網頁版或手機版的 AI 相比有什麼不同嗎?

重點不是模型，而是互動場景

很多用戶會問：我已經買了最貴的方案、用了最好的模型，為什麼還要學電腦端的 AI 工具？

差別在工作流的整合度。網頁版或手機版的 AI，痛點在於每一個斷點都得靠人搬資料 — 每次對話都要重貼一次素材，每次改腳本都要把代碼複製來複製去。任務小、難度低的時候還能接受；任務一複雜，這種做法既撐不久、也擴不大。要做到真正的自動化，得讓 AI 直接接手電腦。今天的課就從安裝開始，一步一步感受電腦端的 AI 怎麼融入日常營運。

網頁版 本地版
任務啟動 整份複製貼上，上下文空間緊張 指定讀取的檔案與段落，可跨多檔案分析
任務完成 需要用戶下載、測試、反饋 AI 直接接手更新、測試、修改
操作軟體 只能文字對答，無法操作軟體 可控制電腦，進行跨軟硬體的操作
客製化 僅能調整對話風格，每次對話重置 可調整單檔風格，並具備跨專案記憶

沒學過程式，能用 AI 寫代碼操作電腦嗎？

95% 的情況下，用自然語言把任務講清楚就夠了，不用學任何程式語言。需要會的，就是打字跟 AI 對話。

前置準備

🛠️ 環境設置：安裝所有需要的工具

📥 第一步：安裝 Claude Code

Windows（PowerShell）： 

irm https://claude.ai/install.ps1 | iex

macOS / Linux： 

curl -fsSL https://claude.ai/install.sh | bash
  1. 安裝完成後，關掉 PowerShell / Terminal 再重新開啟一次，讓 PATH 生效。
  2. 輸入 claude --version，看到版本號就代表裝好了。
  3. 如果安裝或使用過程中遇到問題，可以參考 Claude Code 官方文件。

🟢 第二步：安裝 Node.js

前往 nodejs.org，點擊下方的安裝程式，下載後安裝即可。

🐍 第三步：安裝 Python

前往 python.org/downloads，點擊頁面上最新的 Python 3.14.x 安裝檔下載安裝。

安裝時記得把最下面的 Add python.exe to PATH 打勾，這樣 Claude Code 才能找到 Python。

🔑 第四步：安裝 Tavily

申請 API Key

  1. 打開 app.tavily.com，點 Sign Up 註冊帳號（可以用 Google 帳號直接登入）。
  2. 登入後會看到你的 API Key。在 OPTIONS 中，點左邊第二個圖示就能複製 API Key。
  3. API Key 是你的帳號密碼，不要分享給別人，也不要貼在公開的地方。

裝上 MCP

貼上你的 API Key，下方指令會自動帶入。複製整段到終端機（PowerShell / Terminal）執行： 

claude mcp add tavily --scope user --env TAVILY_API_KEY=tvly-xxxxxxxxxx -- npx -y tavily-mcp@latest

看到 Added stdio MCP server tavily 就成功了。

啟動上手

🖥️ 啟動上手：打開 Claude Code

📂 怎麼打開 Claude Code？

Claude Code 是命令列工具，必須在資料夾裡的終端機啟動。先新建一個空白資料夾，然後接著做下面的說明

Windows：打開資料夾後，按住 Shift 同時點滑鼠右鍵，選 「在這裡開啟 PowerShell 視窗」。

macOS：打開 Terminal，輸入 cd 後把資料夾拖進去，按 Enter 切換到該資料夾。

確認終端機目前的位置是這個資料夾後，輸入： 

claude

畫面會切換成 Claude Code 的對話介面，最下面就是輸入框。

Claude Code 會問你要怎麼登入。選 「Claude account with subscription」，瀏覽器會自動打開 claude.ai 登入頁面，用你訂閱 Pro / Max 的帳號登入即可。

看到 Login successful 就成功了。回到終端機會看到對話介面，輸入 /help 可以看所有可用的指令。

最常做的三件事：中止、重來、還原

Claude Code 會直接動到資料夾裡的檔案，開始之前先把這三個操作放在手邊。

中止 · Esc

AI 跑歪了，按 Esc 立刻打斷。

重來 · /clear

任務完成後，輸入 /clear 清空對話，避免上一輪的脈絡影響下一輪。

還原 · /resume /rewind

任務中間斷掉，用 /resume 或 /rewind 回到上一個狀態，就像遊戲的存檔點。

製作報告

📋 製作報告：做一份品牌分析報告

這節做出第一版報告 — 用 AI 上網查資料，產出 report.html，後面三節會在這份檔案上一層一層加料

✏️ 填上你想分析的品牌 

用 Tavily 針對「___」做一份分析報告。

品牌資訊：
- 品牌名稱：___
- 主營品類：___
- 目前銷售國家：___

請涵蓋以下三個主題，並針對搜到的資料做深入解讀：
1. 品牌概況：公司背景、主要產品線、品牌定位
2. Amazon 表現：熱賣產品、定價策略、評價中看到的優缺點
3. 近期動態與機會：近期新聞、市場趨勢，並給出你的觀察與建議

請著重在「分析與洞察」，不要只是把搜到的資料條列出來，多加入你的推論與串聯。
完成後把報告存成 report.html，用繁體中文撰寫，排版清楚好讀。

工具第一次使用時會跳出權限確認，選 「Yes, and don't ask again for tavily」，之後就不會再問。

完成後，資料夾裡會多一份 report.html，這就是第一版品牌報告。後面幾節會在這份檔案上一層一層加料，最後變成一份可以直接寄給老闆或客戶的完整文件。

讀本地檔案

📁 讀本地檔案：把自家後台數據加進報告

上一節做出第一版報告後，這節把後台 Business Report 疊進去，升級成第二版

讓 Business Report 走進品牌報告

剛剛那份 report.html 全部建立在公開資訊上，少了一塊只有賣家看得到的視角 — Business Report。這份後台報表記錄每個 ASIN 真實的銷售、流量、轉換表現。把它讀進來、跟前面的市場分析對照，報告才開始真正屬於自家品牌。

這也是 Claude Code 比網頁版 AI 強的地方：CSV 不用複製貼上，丟進資料夾就能讀。

先到 Seller Central 下載最近兩個月的 Business Report（按 Parent ASIN），命名為 br-2026-04.csv 與 br-2026-05.csv，放進當前資料夾。再把下面的提示詞貼進輸入框： 

資料夾裡有兩個 Business Report：br-2026-04.csv（4 月）跟 br-2026-05.csv（5 月）。

請幫我做以下分析：
1. 比較兩個月的 Ordered Product Sales，找出下滑最嚴重的 10 個 ASIN
2. 看一下這 10 個 ASIN 的 Sessions 跟 Unit Session Percentage（轉換率）有沒有同步下滑
3. 把結果整理成 markdown 表格，包含：ASIN、4 月 GMS、5 月 GMS、變化幅度、Sessions 變化、轉換率變化、你的初步推論（流量問題還是轉換問題）
4. 把這份分析併進 report.html 的「Amazon 表現」段落，跟前面的市場觀察對照，看結論需不需要修正

注意：CSV 可能很大，必要時請寫一個 Python script 處理，不要把整個檔案讀進對話。

重新打開 report.html，Amazon 表現段落多了真實的 ASIN 銷售數據，跟前面的市場分析互相補完。第二版報告完成。

排版升級

🧩 排版升級：讓報告變成專業文件

第二版報告的內容已經夠扎實，但排版還是濃濃 AI 味。這節用 Skill 補上設計感，產出第三版

🤔 Skill 是什麼？

簡單講，Skill 是一段已經寫好的提示詞。以前大家找 PPT 模板、找 Excel 公式表，到了 AI 時代，找的就是精煉過的提示詞，讓 AI 一次就把一件事做好。

🙌 安裝零門檻

不用寫程式、不用改設定檔，一句話就能裝好。

⚡ 即裝即用

裝完直接生效，不必累積使用記錄才有感。

📐 可被監督

Skill 是純文字檔，可以由人工檢視、判斷內容。

🔎 第一步：先裝一個會找 Skill 的 Skill

既然 Skill 能讓 AI 學會做一件事，那第一個該裝的 Skill，就是讓 AI 知道怎麼找 Skill。find-skills 就是這個入口。把下面這段提示詞貼到 Claude Code 即可： 

全局安裝 find-skills 的 Skill，讓我可以用自然語言直接尋找與安裝其他 Skill
                                REPO Link:https://github.com/vercel-labs/skills/blob/main/skills/find-skills/SKILL.md

裝完後輸入 /skills，如果能在 Skills 清單看到 find-skills，就是安裝成功了。

🎨 第二步：用 find-skills 找一個提升美感的 Skill

前面那份第二版報告內容夠扎實，但排版還是 AI 味很重。這一步找一個能讓 HTML 排版有質感的 Skill。打開 Claude Code，輸入以下提示詞：

本練習統一使用 frontend-design，方便對齊；實際工作中可依需求換成其他 Skill。

幫我找能讓 html 設計變漂亮的 Skill。

📐 第三步：用 frontend-design 重做品牌報告

frontend-design 裝好後，套用到剛剛那份 report.html，就能得到第三版報告 — 內容沒變，但排版變成像樣的商業文件。把下面的提示詞貼到輸入框送出： 

使用 frontend-design Skill，對 report.html 進行以下優化：

- 這份報告要給「___」看
- 設計風格：___
- 加入清楚的資訊層級：標題、小標、重點數據要一眼就能看到
- 數據部分用視覺化的方式呈現（例如卡片、標籤、色塊區分）
- 整體排版要專業、好讀，像一份正式的商業報告

直接修改 report.html，用繁體中文。

重新打開 report.html，視覺會完全不一樣。同樣的內容，第三版報告終於有了專業文件的樣子。

進到後台

🌐 進到後台：讓 AI 直接登入 Seller Central 抓庫存

第三版報告少一塊：庫存健康度。這是後台才看得到的資料，用 Claude for Chrome 把它補上，產出最終版

🤔 Claude for Chrome 是什麼？

Claude for Chrome 是 Anthropic 官方的 Chrome 擴充功能，可以讓 Claude Code 直接操作 Chrome：開分頁、點按鈕、填表單、抓資料。

重點是它會共用你 Chrome 的登入狀態，所以能像本人一樣進入需要登入的後台頁面。

🔐 共用登入

不用重新登入，保留網站登入狀態，Claude Code 直接接手。

👀 看得到畫面

Claude 可以看到瀏覽器的實際畫面，就像用戶本人在操作一樣。

🔁 跨網頁工作流

在 A 網站抓資料、整理後丟到 B 網站，Claude 可以一條龍完成。

📦 安裝 Claude for Chrome 擴充功能

  1. 前往 Chrome Web Store 的 Claude 頁面，點 「加入 Chrome」 安裝擴充功能。
  2. 安裝後會出現在 Chrome 右上角，點開後用 同一個 Claude 帳號登入（就是你用 Claude Code 的那個帳號）。
  3. 回到終端機，關掉現在的 Claude Code（輸入 /exit），然後重新啟動時加上 --chrome 參數： 
claude --chrome

若已經在 Claude Code 裡面，也可以直接輸入 /chrome 啟用連線。連上後，下方狀態列會多一個 Chrome 圖示，代表 Claude 已經能看到你的瀏覽器了。

🔍 實戰：把庫存健康度補進報告

到目前為止，report.html 已經有了市場面（Tavily 抓到的公開資訊）跟銷售面（Business Report 的 ASIN 表現），還缺供應面 — 庫存夠不夠、會不會斷貨、IPI 健不健康。

這些資料只存在 Seller Central 後台。第三方工具看不到，Tavily 也進不去，必須讓 Claude for Chrome 用你的登入狀態去抓。

Step 1：在 Chrome 打開 Seller Central

手動打開 sellercentral.amazon.com，登入到主帳號的首頁。Claude 會用這個分頁的登入狀態進行操作。

Step 2：請 Claude 抓庫存資料

把下面的提示詞貼到輸入框： 

用我目前打開的 Chrome 進入 Seller Central，幫我做以下事情：

1. 從首頁找到 Inventory → Manage FBA Inventory（庫存管理頁面）
2. 抓取目前所有 ASIN 的庫存資料：FNSKU、ASIN、商品名稱、可售庫存、保留庫存、預計到貨
3. 接著進入 Inventory → Inventory Performance Dashboard，把 IPI 分數、過量庫存、超齡庫存、斷貨風險的資料抓下來
4. 整理成兩份 markdown 表格，存成 inventory-snapshot.md

過程中如果遇到驗證碼或登入要求，停下來告訴我，我會手動處理。

Claude 會在 Chrome 裡自己點頁面、抓資料，整個過程都看得到。如果它走錯方向，按 Esc 打斷後，再用「跳過這個」「再點下一頁」這類指令引導它。

Step 3：把庫存資料併進報告

把剛剛抓到的 inventory-snapshot.md 整合進原本的 report.html，產出最終版報告： 

把 inventory-snapshot.md 的內容整合進 report.html，新增一個「供應與庫存健康」段落，重點放在：

- 整體 IPI 分數與斷貨風險
- 過量／超齡庫存佔比
- 跟前面銷售下滑的 ASIN 對照，是不是同一批？是流量問題、轉換問題，還是供應跟不上？
- 給出 2-3 條可執行的建議

直接更新 report.html，保持 frontend-design Skill 的設計風格與排版風格。

到這裡，report.html 從一份公開資料拼湊的初稿，變成一份涵蓋市場、銷售、供應三個維度的完整品牌報告，排版也夠專業。整個過程沒打開 Excel、沒複製貼上，只用了一個資料夾跟幾段提示詞。
